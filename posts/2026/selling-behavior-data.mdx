我花了幾個月把 463 個 xAI 員工的公開行為做成一個儀表板。

然後我嘗試賣它。才發現我搞錯了一件事:沒有人在找一個「追蹤 X 行為的儀表板」。他們在找的,是他們已經有痛點的某件事的答案。這個差距,讓我重新理解了行為數據到底在賣什麼。

---

## 第一:他們付錢買「在做決定前先看清楚」

recruiter 問我「能不能跑一份報告」,不是因為他在找數據工具。是因為他下週要接觸一個人,想先知道:這個人是純潛水仔還是真的有在說話?他的 network 偏哪裡?cold outreach 有沒有機會?

他買的不是儀表板。他買的是「少浪費一次錯誤的 outreach」。

VC 同理。他在看一個 20 人的新創,想知道核心工程師有沒有在 X 上建立存在感、有沒有在公開討論技術方向——還是只有創辦人一個人在對外講話。這個問題值幾千美金,因為後面接著的是幾百萬美金的決定。

**行為數據有價值,不是因為它本身有趣——而是因為它降低了一個後面很貴的決定的不確定性。**

---

## 第二:他們付錢買「自己追不回來的歷史」

這個我一開始沒想清楚。

我的 tracker 每天在跑。一年後,我有一份別人根本不可能重建的時間序資料:誰在哪個時間點開始沉默、誰在某件事發生後活躍度驟升、誰悄悄把 handle 改了。這些訊號不是靜態的,它們只存在於你在「正確時間點」跑的那一次 fetch 裡。

今天有人複製我的 code——他要等一年,才有我現在已有的去年的數據。

很多工具都容易被複製,但「你在正確的時間點開始累積」這件事,不能。護城河不是演算法,是時間序。

---

## 第三:他們付錢買「可以直接交給 agent 跑的訊號」

這個是最新的轉變,也是我現在覺得最值得押注的方向。

當 AI agent 成為正式的工作流程,數據的**形式**開始重要起來。我把 cohort 的行為數據透過 MCP 開放——recruiter 的 agent 可以直接問「這個人最近三個月有沒有在公開討論 infra 相關話題」,拿到答案,不用自己去滑 timeline、過濾、手動整理。

「能不能被 agent 呼叫」,開始決定一份數據能不能嫁接進別人的工作流。這不是技術展示,這是使用情境的改變:數據不再是「你去查」的東西,而是「你告訴 agent 去拿」的東西。

---

我做了幾個月才搞清楚:有人掏錢,從來不是因為他找到一個「有趣的新工具」——而是因為他本來就有一個很痛的決定要做,他追不回來的歷史讓他焦慮,或者他正在試著讓 agent 替他分析而手上沒有可呼叫的資料。

儀表板只是容器。能賣出去的是容器裡那個邏輯接上的痛點。

如果你也在做某種 niche 行為追蹤、或任何形式的「把公開資料做成洞見」的產品——這三個點,可能比你的功能清單更值得先想清楚。（我自己的 dashboard 在[這裡](https://x-watchlist.up.railway.app)——xAI 那份是免費開放的。）

---

**帶刺的一句話:**

如果你的用戶說「很有趣」但沒問價格,你還不在這三件事裡的任何一件。
